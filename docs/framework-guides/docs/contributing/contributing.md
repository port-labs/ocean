# ü§ù Contributing

Thank you for considering contributing to Ocean! This guide will help you understand our development standards and processes.

## Quick Start

1. **Set up your environment**: Follow the [Getting Started](../getting-started/getting-started.md) guide
2. **Review development standards**: Read the sections below to understand our code style, testing, and PR process
3. **Make your changes**: Follow our [Code Style Guide](./code-style.md) and [Testing Standards](./testing-standards.md)
4. **Submit a PR**: Follow our [PR Process Guide](./pr-process.md)

## Development Standards

Before contributing, please familiarize yourself with our development standards:

- **[Code Style & Formatting](./code-style.md)** - Learn about our linting tools, formatting rules, and code quality standards
- **[Testing Standards](./testing-standards.md)** - Understand our testing requirements and best practices
- **[PR Process](./pr-process.md)** - Learn how to create pull requests, branch naming, and PR requirements
- **[Version Management](./version-management.md)** - Understand how to bump versions and create changelogs
- **[Git Workflow](./git-workflow.md)** - Learn about our git workflow, commit messages, and pre-commit hooks

## Types of Contributions

### Report a Bug

If you've found a bug, please:
1. Check if the issue already exists in our [GitHub Issues](https://github.com/port-labs/ocean/issues)
2. Create a new issue with:
   - Clear description of the bug
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (Python version, Ocean version, etc.)

### Suggest a Feature

We welcome feature suggestions! Please:
1. Check existing issues and discussions
2. Create an issue describing:
   - The problem you're trying to solve
   - Proposed solution
   - Use cases and examples

### Develop a New Feature

1. Discuss the feature in an issue first
2. Get approval from maintainers
3. Follow our [development standards](./code-style.md)
4. Write tests following our [testing standards](./testing-standards.md)
5. Submit a PR following our [PR process](./pr-process.md)

### Contribute Documentation

Documentation improvements are always welcome! Please:
1. Follow our documentation style guide
2. Ensure examples are accurate and tested
3. Submit a PR with `[Docs]` prefix in the title

## Getting Help

- **Questions**: Open a discussion on [GitHub Discussions](https://github.com/port-labs/ocean/discussions)
- **Issues**: Check [GitHub Issues](https://github.com/port-labs/ocean/issues)

Please review our [Code of Conduct](https://github.com/port-labs/ocean/blob/main/CONTRIBUTING.md#code-of-conduct) before contributing.
